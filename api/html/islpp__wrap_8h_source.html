222 // Helper methods to pass capturing lambdas into C functions that //
223 // accept a C function pointer with last argument void*. //
225 
226 // NOTE: the number N in isl_wrap_N corresponds to the number of
227 // CALLBACK arguments WITHOUT the trailing void*.
228 //
229 // The functions that accept callbacks are expected to have a void* argument
230 // immediately after each function pointer argument. This argument is supposed
231 // to be passed as the last argument of the callback function without change.
232 // We use this argument to pass a wrapper object that contains an actual lambda
233 // and the original void* pointer provided by the caller. We pass a simple
234 // non-capturing lambda that as C function pointer, which unwraps the wrapper
235 // object and calls the stored potentially capturing lambda.
236 //
237 // NOTE: the wrapper is created as a local stack-allocated variable of the
238 // isl_wrap_N function, and a pointer to it is passed to the C function.
239 // Therefore isl_wrap_N are NOT SUITABLE for DELAYED CALLBACKS, i.e.
240 // callbacks that may be stored and called after isl_wrap_N returns.
241 //
242 // The tuple argument contains the leading arguments of the C function that are
243 // passed unmodified. We enforce strict type matching to avoid hard-to-debug
244 // type conversion problems in template+macro code. The caller of isl_wrap_N
245 // MUST explicitly convert all arguments to the types expected by the C
246 // function, even when implicit conversion is possible, including for void*
247 // arguments.
